# Catch Us Admin Dashboard Layout Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the responsive Catch Us admin dashboard shell, overview layout, and eight navigable mock routes with official shadcn components and an FSD structure that omits the `pages` layer.

**Architecture:** Keep Next.js App Router files in `apps/admin/app` as thin route adapters. Put app-wide providers in `apps/admin/src/app`, route-level UI in `apps/admin/src/widgets`, and business-free navigation configuration in `apps/admin/src/shared`; reusable shadcn primitives remain in `packages/ui`. Use the verified `@shadcn/dashboard-01` block as the layout reference, but install only the official primitives required by the approved static scope so DnD, TanStack Table, Zod, Drawer, and Sonner are not added prematurely.

**Tech Stack:** Next.js 16 App Router, React 19, TypeScript 6, Tailwind CSS 4, shadcn New York/Radix, Recharts, next-themes, pnpm, Turborepo

---

## Scope Guard

This plan implements layout and mock presentation only. Do not add Supabase,
Supabase Auth, uploads, EXIF parsing, TMAP, Geoapify, Google Maps, visibility
mutations, embeddings, face search, or live API calls.

The approved future requirements remain documented in
`docs/superpowers/specs/2026-06-21-admin-dashboard-layout-design.md` and must not
be represented as working functionality in this change.
